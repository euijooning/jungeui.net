## 2. 글 목록(List) UI – 카드 그리드

### 2-1. 카드 그리드 규칙

```
.post-grid {
  display: grid;
  grid-template-columns: repeat(5, 1fr);
  gap: 20px;
}
```

**반응형 컬럼 규칙**

| 해상도 | 컬럼 |
|--------|------|
| ≥1280px | 5 |
| ≥1024px | 4 |
| ≥768px | 3 |
| ≥480px | 2 |
| Mobile | 1 |

- [ ] **4** post-grid 적용, 반응형 컬럼 breakpoint 구현.
- [ ] **5** 페이지당 **5개** 노출, 페이지네이션 UI.

### 2-2. 카드 구조 (고정)

```
┌─────────────────────┐
│ Thumbnail (16:9)    │
├─────────────────────┤
│ Category            │
│ Title (max 2 lines) │
│ Summary (max 3)     │
│ Tags · Date         │
└─────────────────────┘
```

- [ ] **6** Thumbnail 16:9 비율, Category, Title(max 2줄), Summary(max 3줄), Tags · Date.
- [ ] **7** 카드 높이 고정.
- [ ] **8** hover 시: translateY(-4px), shadow-lg.

## 카테고리(사이드바)

- [ ] **9** 사이드바: 카테고리 리스트 (GET /categories).
- [ ] **10** 카테고리 선택 시 해당 카테고리 필터된 목록, **선택된 카테고리명 표시**.
- [ ] **11** 카테고리 미선택 시 "전체" 또는 전체보기.
- [ ] **12** 태그 클라우드(GET /tags) — 선택 사항.

## 데이터 연동

- [ ] **13** GET /posts (status=PUBLISHED). 공개 글만 노출.
- [ ] **14** 카테고리 필터: GET /posts?category= 또는 category_id 반영.
- [ ] **15** 카드 클릭 시 `/posts/:postId` 로 이동.

## 검색 결과

- [ ] **16** 검색은 **제목 기준**, **버튼 클릭 시만** 실행.
- [ ] **17** 검색 결과는 **기존 카드 그리드 재사용** (박스형 리스트 아님).
- [ ] **18** 검색 결과 페이지당 5개 + 페이지네이션 (Phase 05 헤더 검색과 연동).
