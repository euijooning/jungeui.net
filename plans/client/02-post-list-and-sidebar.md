## 라우트 및 레이아웃

- [ ] **1** 포스트 목록 라우트: `/` 와 `/posts` 둘 다 **같은 게시글 목록 페이지**로 연결 (현재 포스트 페이지 = 메인).
- [ ] **2** 레이아웃: 2컬럼 — 좌측 약 70% 목록 / **우측 약 30% 게시판(카테고리)**. 코드그라데이션·sungbin.dev 레퍼런스와 유사하게.
- [ ] **3** 반응형: 좁은 화면에서 사이드바 접기 또는 하단 배치 (선택 구현).

---

## 목록 스타일 및 페이지네이션

- [ ] **4** 목록 스타일: **박스형 리스트** (둥근 모서리 카드 형태).
- [ ] **5** 카드 항목: 썸네일, 카테고리, 제목, 요약, 작성일·태그.
- [ ] **6** 페이지네이션: 목록 **페이지당 최대 5개** 노출, 페이지네이션 UI 제공.
- [ ] **7** 페이지 파라미터: 쿼리(예: `?page=2`) 또는 경로로 페이지 전환.

---

## 카테고리(게시판) 표시

- [ ] **8** 카테고리 선택 시: 게시판(카테고리)이 선택된 경우 **선택된 카테고리명을 화면에 표시** (예: `[기획]/화면설계 심화`). 목록 상단 또는 헤더 근처에 노출.
- [ ] **9** 카테고리 미선택 시: "전체" 또는 카테고리명 영역 비우기/전체보기 문구.
- [ ] **10** 사이드바: 카테고리 리스트 (GET /categories). 대메뉴·소메뉴는 **백오피스 카테고리 관리**에서 등록 후 노출.
- [ ] **11** 카테고리 계층: 현재 categories는 1 depth. 대/소메뉴 필요 시 백오피스에서 parent_id 등 검토 후, 클라이언트는 트리/그룹 표시만 구현.
- [ ] **12** 카테고리 클릭 시 해당 카테고리 필터된 목록 표시 (쿼리 파라미터 또는 경로).
- [ ] **13** 사이드바 추가 요소: 프로필(선택), 태그 클라우드(GET /tags) — client-guide 반영.

---

## 데이터 연동

- [ ] **14** 목록 데이터: GET /posts (status=PUBLISHED). 공개 글만 노출.
- [ ] **15** 카테고리 필터 시: GET /posts?category= 또는 category_id 반영.
- [ ] **16** 카드 클릭 시 `/posts/:postId` 로 이동.

---

## 검색 결과

- [ ] **17** 제목 일치 검색 시 동일한 **박스형 리스트**로 표시.
- [ ] **18** 검색 결과 **페이지당 5개** + 페이지네이션.
- [ ] **19** 검색 실행은 헤더 검색 버튼 클릭 시 (Phase 05 상단 검색과 연동). API에 제목 검색 파라미터(예: title, q) 필요 시 명세 반영.
